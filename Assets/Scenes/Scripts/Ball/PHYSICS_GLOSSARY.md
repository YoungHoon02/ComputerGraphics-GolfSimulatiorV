# 골프공 물리 용어 정리

이 문서는 `BallPhysics.cs` / `PhysicsCore.cs`에서 사용하는 물리 용어와 코드 변수를
1:1로 정리한 것입니다. 시뮬레이션은 **4단계**(비행 → 반발 → 접선마찰 → 구름)로 구성됩니다.

---

## 0. 기본 개념

| 용어 | 의미 | 코드 |
|---|---|---|
| 속도 (velocity) | 공의 3D 이동 속도 벡터 (m/s). Unity 기본 물리가 아닌 **자체 변수**로 관리 | `velocity` |
| 가속도 (acceleration) | 속도를 변화시키는 양. 중력·바람·마찰의 합 | `a_total` |
| Semi-implicit Euler | 적분 방식. **속도를 먼저** 갱신하고 그 속도로 위치를 이동 | `PhysicsCore.Step` |
| FixedUpdate | 고정 시간 간격(`dt`)마다 호출되는 물리 갱신 루프 | `Time.fixedDeltaTime` |

```
v ← v + a·dt      // 속도 먼저
p ← p + v·dt      // 갱신된 속도로 위치
```

---

## 1단계 — 비행 (Flight)

공이 공중에 떠 있는 동안 중력만 받아 포물선을 그리는 단계.

| 용어 | 의미 | 코드 | 기본값 |
|---|---|---|---|
| 중력 (gravity) | 아래로 끌어당기는 가속도 (m/s²) | `gravity` | `9.81` |
| 발사 파워 (power) | 초기 속도의 크기 | `Launch(power, ...)` | 테스트 `12` |
| 발사 각도 (pitch) | 수평면 기준 발사 올림각 (도) | `Launch(.., pitch, ..)` | 테스트 `35` |
| 초기 속도 계산 | 파워·각도 → 속도 벡터 변환 | `CalculateLaunchVelocity` | — |

```
v0 = (forward·cos(pitch) + up·sin(pitch)) · power
```

---

## 2단계 — 바람 (Wind)

비행 중 공에 추가로 작용하는 외력. 역풍이면 비거리가 줄고 순풍이면 늘어남.

| 용어 | 의미 | 코드 |
|---|---|---|
| 바람 가속도 (a_wind) | 바람이 만드는 가속도. `a_total`에 더해짐 | `WindManager.GetWindAccel()` |
| 역풍 (headwind) | 진행 반대 방향 바람 → 비거리 감소 | (Wind X < 0) |
| 순풍 (tailwind) | 진행 방향 바람 → 비거리 증가 | (Wind X > 0) |

```
a_total = a_gravity + a_wind + a_friction
```

---

## 3단계 — 반발 (Bounce / Restitution)

공이 지면에 닿는 **순간**, 속도를 수직·수평 성분으로 나눠 각각 다르게 반사하는 단계.

| 용어 | 의미 | 코드 | 기본값 |
|---|---|---|---|
| 법선 (normal, n) | 충돌 표면이 향하는 방향 (지면은 위쪽) | `n` | — |
| 수직 성분 (v_n) | 속도 중 법선 방향 성분 (튕김에 관여) | `velocity.y` | — |
| 수평/접선 성분 (v_t) | 속도 중 표면을 따라가는 성분 | `velocity.x, z` | — |
| 반발 계수 (e, restitution) | 수직 성분이 튕길 때 남는 비율. `0`=안 튐, `1`=완전탄성 | `*Restitution` | green `0.40` / fairway `0.50` / bunker `0.10` / obstacle `0.60` |
| 접선 마찰 (f) | 바운스 시 수평 속도를 깎는 비율 | `f` | `0.1` |
| 바운스 임계값 | 이보다 빠르게 내려와야 튕김. 느리면 구르기로 전환 | `bounceThreshold` | `0.8` |

```
v_n' = -e · v_n        // 수직: 반발계수만큼 튕김
v_t' = (1 - f) · v_t   // 수평: 접선마찰만큼 감속
velocity = v_n' + v_t'
```

> **e**(반발)는 *얼마나 높이 튀는가*, **f**(접선마찰)는 *바운스마다 앞으로 얼마나 덜 나가는가*를 정합니다.

---

## 4단계 — 구름 (Rolling Friction)

공이 지면에 안착해 구르는 동안 점점 감속하다 멈추는 단계.

| 용어 | 의미 | 코드 | 기본값 |
|---|---|---|---|
| 구름 마찰 계수 (μ, mu) | 표면별 구름 저항. 클수록 빨리 멈춤 | `*Friction` | green `0.45` / fairway `0.85` / rough `1.4` / bunker `2.2` |
| 마찰 가속도 | 진행 반대 방향 등속 감속 | `GetFrictionAccel` | — |
| 정지 임계값 | 이보다 느리면 0으로 처리(무한 미끄러짐 방지) | `stopThreshold` | `0.1` |
| 접지 상태 | 공이 지면에 얹혀 있는지 | `isGrounded` | — |

```
a_friction = -μ · g · v̂      // v̂ = 속도 방향 단위벡터
```

마찰 가속도가 일정하므로 속도는 **선형으로 감소**하고, 이동 거리는 정지에 가까워질수록 완만해집니다.

---

## 표면 종류 (SurfaceType)

표면마다 반발 계수 `e`와 구름 마찰 `μ`가 다릅니다.

| 표면 | 반발 e | 마찰 μ | 특징 |
|---|---|---|---|
| Green (그린) | 0.40 | 0.45 | 잘 구름, 덜 튐 |
| Fairway (페어웨이) | 0.50 | 0.85 | 표준 |
| Rough (러프) | 0.50 | 1.4 | 빨리 멈춤 |
| Bunker (벙커) | 0.10 | 2.2 | 거의 안 튀고 푹 박힘 |
| Obstacle (장애물) | 0.60 | — | 잘 튕김 |
| OutOfBounds (OB) | — | — | 벌타·리셋 |

표면 판정은 충돌 레이어를 읽어 결정합니다: `LayerToSurface()`.

---

## 상태 전이 한눈에

```
발사(Launch) ──▶ [1·2단계 비행] ──지면 도달──▶ [3단계 반발]
                                                  │
                          내려오는 속도 > bounceThreshold ? 
                          │ 예 → 다시 튕겨 비행으로                 
                          │ 아니오 → [4단계 구름]                  
                                                  │
                              속도 < stopThreshold ──▶ 정지
```
